## 3. 제품 철학 (Design Principles)

1. **보험은 손실 이전 장치다**: 감당 불가능한 손실을 감당 가능한 수준으로 옮기는 것. 아파서 돈 버는 장치가 아니다. 감당 가능한 손실은 내 돈으로 처리하는 게 원칙.
2. **줄이라고 말할 수 있다** (D1): 결과가 "이 보장은 빼도 된다"여도 그대로. 성과는 사용자 주머니에 남은 돈으로 측정.
3. **진지하되 지루하지 않게** (D3′): 신뢰는 진지함에서, 완주는 리듬에서 나온다. 과정을 잘게 쪼개고(퀘스트), 단계마다 작은 결과를 돌려준다(마이크로 리빌). 오락화는 하지 않는다.
4. **가이드하되 강요하지 않는다** (D4): 시스템은 제안하고 결정은 사용자가. 조정하면 결과가 즉시 바뀌는 것을 보여준다.
5. **낮은 문턱, 높은 천장** (D5): 30초 입력으로도 의미 있는 결과를 주고, 더 정확한 답을 원하는 만큼만 입력을 늘리게 한다. 어떤 데이터도 "시작의 조건"으로 요구하지 않는다.
6. **가정은 전부 공개·조정 가능**: 물가·갱신 인상률 등 기본값을 깔되, 기본 화면에는 노출하지 않고 "계산 근거 보기"에 격리 (D2).
7. **불확실성은 범위로**: "무난한 경우 / 최악의 경우" 두 세계. 단일 숫자로 겁주지 않는다.

## 4. 핵심 프레임: "보험 검진"

### 4.1 왜 검진 문법인가

```
문진 (내 상황 입력) → 검사 (위험 시나리오 점검) → 결과지 (보장 상태 판정)
→ 처방 (원 단위 3줄) → 정기 재검진 (연 1회 / 라이프이벤트 시)
```

- **진지함과 대중성을 동시에 만족**: 대중은 검진 결과지 읽는 법을 이미 안다 — 정상/주의/이상 3색, 항목별 판정, 처방. 학습 비용 제로, 톤은 진중 (D2·D3′).
- **"줄여라"가 자연스러운 문법**: 검진에서 "이 약은 그만 드셔도 됩니다"는 신뢰받는 소견이지 영업 멘트가 아니다 (D1).
- **간이→정밀의 2단이 원래 검진의 문법**: 간이 문진으로 시작해 필요하면 정밀 검사를 받는 구조가 자연스럽다 (D5).
- **재방문이 내장된 프레임**: 검진은 원래 정기적으로 받는 것.

### 4.2 판정 체계 (결과지의 뼈대)

보장 영역(암·뇌심혈관·사망·장해·의료비·소득상실)별로 검진식 판정:

| 판정 | 의미 (내부 기준) |
|------|----------------|
| 🟢 **적정** | 최악 시나리오 손실이 [감당 한도 + 보유 보장] 이내 |
| 🔴 **부족 (빈 곳)** | 최악 시나리오가 한도를 초과하는데 보장이 없거나 모자람 |
| 🟡 **과잉 (넘치는 곳)** | 감당 한도 이내의 손실에 보험료를 내고 있음 — 소액 특약 등 |
| 🟡 **중복 (겹치는 곳)** | 동일 위험에 복수 계약이 겹침 |
| ⚪ **미검진** | 데이터 부족으로 판정 보류 — 추가 문진 퀘스트로 연결 |

모든 판정에는 **정확도 게이지**가 따라붙는다 (§5.3) — 입력 수준에 따라 판정의 신뢰 수준을 정직하게 표시.

## 5. 사용자 경험 설계

### 5.1 대원칙: 입력은 계단식, 결과는 즉시

- **문항은 퀘스트가 필요로 할 때만 등장한다 (지연 문진).** 긴 폼을 앞에 두지 않는다.
- **어느 계단에서 멈춰도 결과지는 발급된다.** 정확도만 달라진다.
- 연동은 두 가지 이유가 생겼을 때만 권유된다: ① 정확도를 올리고 싶다 ② 놓친 보험금 스캔을 하고 싶다(연동 없이는 물리적으로 불가능한 유일한 기능).

### 5.2 정밀도 사다리 (Precision Ladder) — D5의 실체

| 레벨 | 입력 | 소요 | 무엇이 가능한가 | 계산의 기반 |
|------|------|------|----------------|------------|
| **L0. 맛보기** | 나이·성별·연소득 (3문항) | 30초 | 위험 점검 시작, 첫 가판정 | **"같은 조건 평균"** — 동연령·소득대 통계 프로필 (평균 자산·평균 보유 보장 구성) |
| **L1. 내 상황** | + 모아둔 돈, 가구 구성(외벌이/맞벌이·부양가족) | +1분 | "버틸 수 있는 금액"이 내 것이 됨 → 판정 개인화 | 통계 프로필 중 자산·가구만 실측 치환 |
| **L2. 내 보험** | + 월 보험료 총액, 주요 보장 유무·금액 체크 (실손? 암 진단비 얼마? 사망보장 얼마? — **아는 만큼만**, 모르면 "모름") | +1~2분 | 보유 보장 반영 판정 — 넘침·겹침 잠정 판정 가능 | 보장은 자기 신고 기반 |
| **L3. 정밀 (연동)** | 마이데이터 연동 | 인증 1회 | **전 계약·특약 자동 반영** (자기 신고 오류 교정) + **놓친 보험금 스캔 해금** + 발생률 개인화(건강검진 데이터) | 실데이터 |

**설계 의도:**
- L0의 "같은 조건 평균" 장치가 핵심이다. 아무것도 모르는 상태에서도 *"35세 남성, 연소득 6,000이라면 보통 월 ○원의 보험료를 내고, 이런 구성을 갖고 있어요. 이 평균적인 사람의 위암 검진 결과는 —"* 를 보여줄 수 있다. **남의 결과(평균)를 먼저 보여주면, 내 결과가 궁금해진다** — 이것이 L1으로 올라가는 자연 동력.
- L2의 "아는 만큼만" 원칙: 보험을 모르는 게 부끄럽지 않게. "모름"을 누르면 그 항목은 평균으로 두고 ⚪미검진 대신 "추정" 표시.
- L3 연동의 이중 유인: **정확도 최대화** (자기 신고를 실계약으로 교정 — "기억과 실제는 다릅니다") + **놓친 보험금 스캔 해금** (연동 필수 기능이자 즉시 현금 보상). 연동을 "요구"가 아니라 **퀘스트 보상 해금**으로 표현한다.

### 5.3 정확도 게이지

- 결과지·판정 카드에 상시 표시: `정확도 ●●○○○ — 같은 조건 평균 기반` / `●●●●○ — 내 보험 반영 (자기 신고)` / `●●●●● — 실제 계약 반영`.
- 게이지 터치 → "정확도를 올리려면" = 다음 계단 문항 or 연동. **CTA가 아니라 게이지가 연동을 판다.** 정직한 한계 고지가 곧 업셀 장치.

### 5.4 퀘스트 구조 (D3′)

| 퀘스트 | 입력 (지연 문진) | 완료 즉시 보상 (마이크로 리빌) |
|--------|----------------|------------------------------|
| Q1. 기본 문진 | L0 3문항 | "당신과 같은 조건의 평균 재정 방어선" 표시 |
| Q2. 위험 점검 ① 암 | 없음 (L0 기반) | 암 영역 **가판정** — "같은 조건 평균이라면 🔴" |
| Q3. 내 상황 반영 | L1 문항 | **"당신이 버틸 수 있는 금액: 약 ○○만원"** (제안값+조정 슬라이더) → Q2 판정이 즉시 갱신되는 걸 보여줌 |
| Q4. 내 보험 반영 | L2 문항 (아는 만큼만) | 판정 2차 갱신 + "월 보험료 ○원 중 일하고 있는 보장은?" |
| Q5. 위험 점검 ②③ | 없음 | 뇌심혈관 / 사망·소득상실 판정 카드 |
| Q6. 종합 결과지 | — | 결과지 + 처방 3줄 + 정확도 게이지 |
| Q7. 정밀 검진 (선택) | **마이데이터 연동** | 전 계약 자동 반영(신고 오류 교정 리포트) + **놓친 보험금 스캔 해금 → "청구 가능 추정 ○○원"** |
| Q8. 처방 실행 | — | 청구/정리/보강 각각 "회수·절감 ○원" 확정 |

- **진행 구조**: 검진 항목 체크리스트 UI. 완료 🟢 / 미완료 ⚪ "미검진". 부분 완료여도 결과지 발급 (§5.1) — "미검진 3개·정확도 ●●○○○"가 자연스러운 재방문 훅.
- **리워드는 정보와 돈**: 포인트·배지 없음. 가장 강한 보상은 "내 돈이 어디서 새는지"라는 정보 자체. (가벼운 완료 체크 애니메이션 정도는 OK)

### 5.5 진입 — 시작 문턱 제로

- 진입 질문(안): **"내 보험, 일하고 있을까요? — 30초면 시작합니다"** / "청구 안 한 보험금이 있는지 확인해 보세요"
- 로그인·연동·앱 설치 없이 웹에서 L0 즉시 시작 (열린 질문 Q3). 연동 이야기는 결과지를 받기 전까지 등장하지 않는다.

### 5.6 용어 번역표 (D2 — 표면에서 전문 용어 금지)

| 금지 (내부 전용) | 사용자 표면 |
|----------------|-----------|
| 리스크 캐패시티 | **버틸 수 있는 금액** |
| 손실 분포 p50 / p90 | **무난한 경우 / 최악의 경우** |
| 자가보험(self-insure) | **보험 없이 감당해도 되는 범위** |
| 보장 갭 / 과잉 / 중복 | **빈 곳 / 넘치는 곳 / 겹치는 곳** |
| TCO·NPV·기회비용 | **이 보험의 진짜 가격** / "이 돈을 그냥 모았다면" |
| 실질가치 (인플레 조정) | **그때 돈으로 얼마** ("2036년의 1억 = 지금 돈 약 6,700만") |
| 산정특례 | **암은 건강보험이 95%를 부담하는 제도** |
| 갱신형 / 비갱신형 | **나중에 비싸지는 보험 / 처음부터 고정인 보험** |
| 소득상실 | **끊기는 월급** |
| 통계 프로필(코호트 평균) | **같은 조건 평균** ("당신과 같은 나이·소득대라면 보통—") |

원칙: 용어 순화가 아니라 **개념을 사건으로 서술**. ("소득상실이 발생합니다" ✕ → "치료하는 8개월간 월급이 끊깁니다" ○)

### 5.7 "버틸 수 있는 금액" 조정 UX (D4)

- 시스템 제안값 + 근거 한 문장: "모아둔 돈 3,000만원에서 만약을 위한 여윳돈 20%를 빼고 잡았어요."
- 슬라이더/직접 입력으로 조정 → **모든 영역 판정이 실시간 재계산** (조정하면 🔴가 🟢로 바뀌는 게 보인다).
- 제안값 복원 버튼 상시. 어떤 값을 골라도 판단하지 않는다.

### 5.8 시나리오 점검의 진행 방식 (지루함 방지)

위험 점검 퀘스트는 긴 설명 대신 **짧은 문답 리듬**. 게임 연출은 없지만 한 방에 결과를 던지지도 않는다.

1. **원탭 예측**: "위암 치료비, 본인 부담이 얼마쯤 될 것 같나요?" — 3택 원탭.
2. **사실 공개**: "약 ○○만원. 암은 건강보험이 대부분을 부담합니다(본인부담 5%)."
3. **핵심 반전 제시**: "진짜 부담은 치료비가 아니라 **치료 기간 동안 끊기는 소득**입니다. 8개월 기준 약 ○○만원."
4. **내 상황 대입**: 버틸 수 있는 금액과 대조 → 판정 카드 발급.

각 스텝 1화면, 영역당 4~6화면. 스크롤 리포트가 아니라 **카드 넘김 리듬**.

### 5.9 톤 & 금지선

- **기준 톤**: 검진 결과를 설명해 주는 신뢰할 수 있는 전문가 — 담담하고, 쉽고, 판단을 강요하지 않음.
- **금지**: 공포 소구, 오락화(질병 캐릭터화·게임 연출·경품성 표현), 확정적 예언. 시나리오는 항상 "만약 ~라면".
- 실제 유병자·경험자 분기: 시나리오 점검 대신 **놓친 보험금 스캔·보유 계약 진단** 중심 동선.
- 사망 시나리오는 "남은 가족의 통장" 관점으로 서술.

### 5.10 결과지 & 처방 — 반드시 돈으로 끝난다 (D1)

**보장 검진 결과지** (Q6):

- 상단: 종합 소견 한 문장 + **정확도 게이지**.
- 몸통: 영역별 판정표 (🟢/🔴/🟡/⚪) + 항목별 한 줄 소견.
- **하단 처방 3줄 (전부 원 단위):**
  1. 💰 "청구 안 한 보험금 **○○원** — 지금 청구하세요" (L3 해금 시. 미연동자에겐 "스캔하려면 정밀 검진" 안내)
  2. ✂️ "이 특약 2개는 버틸 수 있는 범위입니다 — 정리 시 **월 ○원** 회수" (+ 해지 전 재가입 불가 리스크 자동 병기)
  3. 🛡️ "소득 공백 대비가 비어 있습니다 — 채우면 **월 ○원**" (빈 곳이 실재할 때만 등장)
- 결과지는 저장·PDF·(선택) 공유. 공유본은 판정 등급만, 금액·개인정보 제외.
- 재검진 훅: 연 1회 + 라이프이벤트(결혼·출산·이직·주담대) 시 "재검진 권장" 알림.

## 6. 계산 개념 모델 (내부 엔진 — 표면 비노출)

### 6.1 리스크 캐패시티 = "버틸 수 있는 금액"
```
기본 제안값 = 유동자산 + 위기 기간 중 유지되는 가구 소득 여력 − 필수 고정지출 − 안전 마진(기본 20%)
```
L0에서는 코호트 평균으로 대체, L1부터 실측 치환. 사용자 조정 가능 (D4). 조정값이 모든 판정의 기준선.

### 6.2 이벤트 손실 모델
```
총손실 = 직접의료비 [급여 본인부담(산정특례 반영) + 비급여] + 소득상실 + 간병·부대비용
       − 공적보장(건강보험·산정특례·상병수당) − 보유 보장(실손+정액)
```
급성기/치료기/회복기 타임라인. 표면 출력은 무난(p50)/최악(p90) 2트랙 (내부 3점).

### 6.3 적정 보장액 (판정 로직)
```
빈 곳 = 최악의 경우 손실 − 버틸 수 있는 금액(조정값) − 보유 보장
```
≤0 → 🟢 (보유 보장이 한도 안 손실에 붙어 있으면 🟡 넘침). >0 → 🔴. 소액 보장엔 **청구 누락 확률 페널티** 반영 (P2).
**입력 레벨별 신뢰도**: 보유 보장 항이 코호트 평균(L0)·자기 신고(L2)·실계약(L3) 중 무엇인지가 정확도 게이지의 원천.

### 6.4 TCO = "이 보험의 진짜 가격"
```
진짜 가격 = 보험료 흐름 NPV + 기회비용(차액 투자 시 축적액) − 기대 보험금의 실질가치(인플레 조정)
          ± 재가입 리스크 (갱신 인상률 + 유병자 가입거절 확률)
```
**핵심 비대칭 (정직하게 노출)**: 갱신형의 진짜 리스크는 보험료 인상이 아니라 **재가입 불가능성**(나중에 아픈 몸으로는 새 보험을 못 든다). 비갱신 장기의 진짜 리스크는 **물가로 인한 보장가치 침식 + 중도 해지 매몰**. 어느 쪽이 항상 이기지 않는다 — 답이 개인마다 다르다는 것이 이 도구의 존재 이유.

### 6.5 코호트 추정 엔진 (L0/L1의 기반 — 신규)
- 연령×성별×소득분위별 **통계 프로필**: 평균 유동자산, 가구 구성 분포, 평균 보험료 지출, 보유 보장 구성(실손 가입률, 평균 진단비 등).
- 소스: 통계청 가계금융복지조사, 보험연구원 보험소비자 조사, 보험개발원 가입 통계 + (축적 시) BOMAPP 자체 익명 통계.
- 표면 표현은 항상 **"같은 조건 평균"** — 개인 예측처럼 말하지 않는다 (정확도 게이지와 한 몸).

## 8. 대표 플로우 — "위암 위험 점검" (정밀도 사다리 반영)

> ⚠️ 수치는 구조 설명용 가정치. 페르소나: 35세, 세후 월 500만, 외벌이. **연동 없이 진행.**

1. **[진입]** "내 보험, 일하고 있을까요? — 30초면 시작합니다" → L0 3문항 (나이·성별·연소득).
2. **[Q2 가판정]** "당신과 같은 조건(30대 중반·연소득 6천)의 평균이라면 — 위암 시 최악 -8,700만원, 평균 방어선 2,100만원. **평균적인 사람은 🔴 뚫립니다.** 당신은 어떨까요?" ← 남의 결과가 내 결과를 궁금하게 만든다.
3. **[Q3 내 상황]** L1 2문항 (모아둔 돈 3,000만·외벌이) → **"당신이 버틸 수 있는 금액: 약 2,400만원"** + 조정 슬라이더. 판정 실시간 갱신.
4. **[위험 점검 문답]** 원탭 예측("치료비 얼마쯤?") → 사실 공개("약 400만원 — 건강보험이 95% 부담") → 핵심 반전("진짜 부담은 8개월 끊기는 월급 약 4,000만원").
5. **[Q4 내 보험]** "실손 있어요?(예) 암 진단비 있어요?(없음) 월 보험료 총액은?(23만원)" — 아는 만큼만.
6. **[판정]** 🔴 **빈 곳: 소득 공백 대비 약 6,000만원 부족.** 병원비 쪽 🟢. "월 23만원 중 소액 특약 추정분은 🟡 넘치는 곳일 수 있어요 (정확도 ●●●○○ — 자기 신고 기반)."
7. **[결과지]** 종합 소견 + 처방: ✂️"소액 특약 정리 검토 — 월 ○원 회수(정밀 확인 권장)" 🛡️"소득 공백용 일시금 6,000만원 — 월 ○원". 💰 자리에는: **"청구 안 한 보험금이 있는지는 실제 계약·진료 이력이 필요해요 — 정밀 검진으로 스캔 해금"**.
8. **[Q7 연동 선택]** 여기서 처음 연동 등장. 유인 = 정확도 ●●●●● + 놓친 보험금 스캔. 거절해도 결과지는 온전히 소유.

**이 플로우가 파는 것**: ① "병원비는 이미 막혀 있고 진짜 구멍은 월급"이라는 반전 (지루하지 않음의 원천) ② 연동 없이 여기까지 온 뒤, **연동이 '요구'가 아니라 '더 받고 싶은 것'이 되는 순간**.

| **비연동 서비스의 데이터 처리** | L0~L2 자기 신고 데이터의 성격 (마이데이터 밖) | 수집·보관 최소화 원칙, 개인정보 처리방침 별도 정리 |
| **코호트 통계 활용** | 자체 사용자 통계의 비식별 활용 (M0) | 비식별 조치·법무 검토 전제 |
| 건강 데이터 기반 예측 | 의료행위 오인 방지 | "통계적 추정, 진단 아님" 고지. "검진" 명칭의 의료 오인 여부도 확인 |
| 판정·소견 표현 | 검진 문법("소견·처방")의 책임 범위 | 면책 문구·표현 수위 법무 검토 |

## 12. 로드맵

### 실 서비스 트랙

**Phase 1 — 간이 검진 (완전 비연동, L0~L2)**
- M0 코호트 추정 엔진 + Q1~Q6 전체: 지연 문진, 암 1종 위험 점검, 자기 신고 보험 반영, 결과지+정확도 게이지.
- **연동 기능 일절 없음** — 이 단계 자체로 완결된 서비스. 검증: 완주율, 계단 상승률(L0→L1→L2), "정밀 검진 원함" 의향률.
- 상세 기획: [insurance-adequacy-simulator-phase1-spec.md](./insurance-adequacy-simulator-phase1-spec.md) (화면 플로우 13화면·계산 스펙·데이터 소싱·기술 배치 초안)

**Phase 2 — 정밀 검진 (L3 연동 + 놓친 보험금 스캔)**
- 마이데이터 연동: 전 계약 자동 반영 + 자기 신고 교정 리포트 ("기억하신 것과 실제 계약이 3곳 달랐어요").
- **M6 놓친 보험금 스캔 동시 출시** — 연동의 킬러 유인이므로 연동 기능과 한 몸으로 나가야 함 (분리 시 연동할 이유가 "정확도"뿐이라 약함).
- M5(실질가치)·전 영역 점검(뇌심혈관·사망) 확장.

**Phase 3 — 정기화·개인화**
- 연 1회 재검진 + 라이프이벤트 트리거, 질병위험예측 결합 발생률 개인화(L3), M4 비교기 고도화.

### 마케팅 트랙 (별도 — 별첨 A)
- HP/데미지 바이럴 시뮬레이션, "그때 돈으로 얼마?" 단독 계산기, "놓친 보험금" 티저. 실 서비스 Phase 1 안착 후 유입 채널로 가동. 캠페인 랜딩은 항상 간이 검진(L0)으로.

## 14. 열린 질문 (v0.4)

~~해결됨: 타깃(대중) · 중립성(줄이라고 말한다) · 감당선(가이드+조정) · 재미 수위(진지+퀘스트) · 게임 프레임(마케팅 보관) · 연동 시점(시작 조건 아님 — 정밀도 사다리)~~

1. **"검진" 프레임 확정**: 건강검진 문법을 마스터 프레임으로 확정? (의료 오인 리스크 법무 확인 — 대안 명칭: "보장 점검", "보험 진단")
2. **L0 "같은 조건 평균"의 데이터 전략**: 공개 통계 조합으로 시작할지, 자체 마이데이터 익명 통계(신선하지만 법무 검토 필요)를 처음부터 쓸지.
3. **포지셔닝**: 간이 검진(L0~L2)은 앱 설치 없는 웹 진입이 유리해 보임. 정밀 검진(L3)에서 본앱 합류 — 이 웹→앱 동선에 동의하는지.
4. **자기 신고(L2)와 실계약(L3)의 충돌 처리**: 연동 후 "기억과 실제가 다름" 교정 리포트를 어느 수위로 보여줄지 (교정 자체가 강한 각성 순간이자, 자칫 무안 주기가 될 수도).
5. **사망 시나리오 포함 시점**: Phase 1부터(종신보험 P5·P6 직결) vs 암으로 시작 후 확장(소재 무게).
6. **B2B2C**: 검진 엔진을 Wings(설계사 도구)에 제공 — "정직한 설계사의 무기" 경로. 판매 조직과의 긴장을 감수할 것인가.

## 별첨 A — 마케팅·바이럴 아이디어 보관 (v0.2 원안, 실 서비스 미적용)

> D3′ 결정으로 실 서비스에서 제외. 추후 유입 캠페인용으로 보관. 캠페인 랜딩은 항상 실 서비스 간이 검진(L0)으로 연결한다.

### A.1 HP/방어구 데미지 시뮬레이션
- 콘셉트: "5년 뒤 큰 병에 걸린 나, 통장은 버틸 수 있을까?" — 체력바(버틸 수 있는 돈)가 데미지(치료비+끊긴 월급)에 깎이는 3~5분 시뮬레이션.
- 게임 문법 매핑: HP=리스크 캐패시티, 데미지=이벤트 손실, 기본 방어=건강보험, 방어구=보유 보험, 빈 틈=보장 갭, 과무장=과잉 보장.
- 퍼널: Hook(3문항 즉시 시작) → 이벤트 발생 → 예측 게임 → 마이크로 리빌 ①"병원비는 나라가 95% 막아줌" → ②"진짜 데미지는 8개월 끊긴 월급" → HP 정산 → 공유 카드("나는 최악의 시나리오에서 8개월 버티는 사람").
- 가치: 이중 반전 구조가 공유 동기. "과무장 해제"가 게임 문법에서는 자연스러운 합리적 행동이라 D1 메시지와 정합.
- 리스크: 질병 소재 오락화 역풍 — 담담한 톤 유지, 유병자 우회 분기 필수.

### A.2 "그때 돈으로 얼마?" 단독 계산기
- "10년 전 가입한 종신보험 1억, 지금 돈으로 얼마일까?" — 입력 2개(가입연도·가입금액)로 즉답하는 초경량 위젯. 제작비 대비 공유 효율 최상. 결과 화면에서 간이 검진(L0)으로 유도.

### A.3 "청구 안 한 보험금" 티저
- "한국인 1인당 놓친 보험금 평균 ○○원" 류의 통계 티저 → 정밀 검진(놓친 보험금 스캔)으로 직행 랜딩. Phase 2 출시와 동기화.
